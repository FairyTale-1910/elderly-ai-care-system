# 💬 ChatController 核心对话接口文档

`ChatController` 是智慧老人陪护系统中最核心的交互控制层。它负责处理前端用户的文本对话输入，协调大语言模型（LLM）生成回复，解析特殊功能标签（情绪监测、日程提醒），并最终调用科大讯飞引擎进行语音合成（TTS）。

---

## 1. 核心会话接口 (发送聊天消息)

该接口用于接收前端传来的老人对话文本，处理上下文记忆，解析 AI 的特殊指令，并返回经过处理的文本和语音流。

- **请求路径**: `/api/chat/send`
- **请求方式**: `POST`
- **Content-Type**: `application/json`

### 📥 1.1 请求参数 (Request)

**URL 参数 (Query String)**
| 参数名 | 类型 | 必填 | 描述 |
| :--- | :--- | :--- | :--- |
| `uid` | Long | **是** | 当前登录老人的唯一用户 ID。 |

**请求体参数 (Body - JSON)**
| 参数名 | 类型 | 必填 | 描述 |
| :--- | :--- | :--- | :--- |
| `content` | String | **是** | 老人本次发送的文本内容。 |
| `sessionId` | String | 否 | 对话会话 ID。不传则自动新建；若提供，则继续加载该会话的历史记忆。 |

**请求体示例:**
```json
{
  "content": "我这几天心里总是慌慌的，睡不好觉。",
  "sessionId": "a1b2c3d4" 
}
```

### 📤 1.2 响应参数 (Response)

成功调用后，后端将返回带有文本、状态、会话 ID 以及语音流的 JSON 数据。
| 参数名 | 类型 | 必填 | 描述 |
| :--- | :--- | :--- | :--- |
| status | String | 请求状态 | success 或 error |
| response | String | AI 助手剥离内部标签后的纯文本回复 |
| session_id | String | 当前会话 ID |
| emotion | String | 识别到的负面情绪（无则为 null）|
| audioBase64 | String语音合成的 Base64 编码字符串 | 
| message | String | 错误信息（仅发生异常时返回）

## 2.执行流程详解 (内部机制)

### 阶段 1：参数获取与会话管理
从请求体获取用户输入内容 content。

判断 sessionId：

不存在：生成 8 位 UUID 作为新会话 ID。

存在：沿用旧会话 ID，保持多轮对话上下文。

### 阶段 3：读取历史对话（记忆功能）
查询当前会话最近 10 条记录，按时间正序排列，作为 AI 上下文。

性能优化：仅取最近 10 条，平衡上下文完整性与性能，避免记忆过长导致冗余。

排序逻辑：子查询倒序取最新，外层正序排列，保证 AI 接收的上下文顺序正确。

### 阶段 4：调用 AI 获取回复
将整理后的对话历史（用户 + AI 过往消息）传入 AIService，接收 AI 返回的带功能标签的原始回复：

### 阶段 5：标签解析（核心扩展功能）

#### 5.1 情绪标签解析 [EMOTION]
提取 [EMOTION] 与 [/EMOTION] 之间的情绪内容

擦除标签：保留干净的回复文本（cleanReply），用于展示和语音合成。

#### 5.2 提醒任务解析 [REMINDER_TASK]
[REMINDER_TASK]任务名|完整时间|重复频率[/REMINDER_TASK]

提取标签内内容，按 | 拆分任务名、完整时间、重复频率

转换重复频率为数据库存储格式（0 = 一次、1 = 每日、2 = 每周）

拆分完整时间为 “时分”（remind_time）和 “完整时间”（next_remind_time）

执行 SQL 将提醒任务存入 reminders 表，自动创建老人提醒

移除标签，保留干净的回复文本
